# Code of Conduct

## The standard

Treat contributors with patience and professional respect. Give specific feedback, explain the
reasoning behind a correction and keep disagreement focused on the work.

Harassment, threats, sexualized attention, deliberate intimidation, insults, trolling and the
publication of somebody else's private information are not acceptable here.

Use reproducible mathematical examples when reporting a problem. Do not post credentials,
private documents or screenshots containing data that does not belong to you.

## Scope and enforcement

This code applies to issues, pull requests, reviews and other spaces where somebody is
representing IntegraDraw. Maintainers may edit or remove content, reject a contribution, issue a
warning or exclude somebody from project spaces when these standards are ignored.

Report a conduct concern privately to [info@ejupilabs.com](mailto:info@ejupilabs.com). Report a
software vulnerability through
[GitHub private vulnerability reporting](https://github.com/ejupi-djenis30/IntegraDraw/security/advisories/new).
Reports will be handled confidentially, and retaliation for a good-faith report is not
acceptable.
